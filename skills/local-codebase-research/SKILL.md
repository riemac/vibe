# 本地代码库调研

## 核心工具：语义搜索

`mcp_augmentcode_codebase-retrieval` 是调研的核心工具，适用于绝大多数场景。

### 为什么优先使用

| 特性 | 说明 |
|------|------|
| **意图驱动** | 用自然语言描述想了解什么，无需预知关键词或文件位置 |
| **跨边界关联** | 一次调用覆盖多项目、多文件、多层级（README ↔ 代码 ↔ 配置） |
| **一次到位** | 返回代码片段 + 文件路径 + 上下文，减少后续调用 |

### 适用范围

语义搜索适用于**绝大多数调研场景**，包括：

- **探索性调研**：不知道代码在哪、不知道该搜什么关键词
- **定位性调研**：知道概念但不知道实现位置
- **理解设计思想**：需要跨文件关联理解架构
- **查找可复用代码**：找现有实现作为参考
- **理解 API 用法**：查找调用示例、配置方式和源代码
- **编码前调研**：理解现有实现再动手改

### 查询技巧

**使用长句，描述完整意图（6W3H/5W2H 式）**：

```
"[项目/模块] 的 [核心概念] 是什么？
 它解决了什么问题？使用了什么方法？
 输入输出是什么？如何实现 [具体技术点]？"
```

### 查询示例

```
# 全面了解项目
"TRO-Grasp 的完整技术架构是什么？图表征如何设计？
 节点和边的含义？扩散模型如何工作？如何实现跨手型泛化？"

# 查找可复用代码
"在灵巧手仿真环境中如何获取物体点云？是相机还是直接采样？
 有哪些现成的 API？预处理流程是什么？"

# 理解 API 用法
"Isaac Lab 中如何配置深度相机传感器？数据类型有哪些？如何获取深度图？"

# 跨项目对比
"Get-Zero 和 TRO-Grasp 如何实现跨手型泛化？观测表示有什么不同？"
```

## 辅助工具

### 精读文件

语义搜索后，如需补充完整上下文，用 `read_file` 或 `mcp_filesystem-mc_read_content`（批量加速，优先）精读

### 精确匹配

`grep_search` **仅用于**：找符号的所有引用（如重构前找调用点）

```python
grep_search(query="class EmbodimentTransformer", isRegexp=False)
```

---

## 原则

1. **事实驱动**：引用具体文件路径和代码片段作为证据
2. **并行加速**：多个独立查询可并行执行
3. **证据不足时继续检索**，不猜测
